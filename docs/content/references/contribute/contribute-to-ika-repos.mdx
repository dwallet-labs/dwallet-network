This page describes how to contribute to Ika, and provides additional information about participating in the Ika community.

## Join the community {#join-the-community}

To connect with the Ika community, join our [Discord](https://discord.gg/ika).

## Open issues {#open-issues}

To report an issue with Ika, [create an issue](https://github.com/MystenLabs/sui/issues/new/choose) in the GitHub repo. Click **Get started** to open a template for the type of issue to create.

## Fork Ika to contribute {#fork-ika-contribute}

To contribute to Ika source code or documentation, you need only a GitHub account. You can commit updates and then submit a PR directly from the Github website, or create a fork of the repo to your local environment and use your favorite tools to make changes. Always submit PRs to the `main` branch.

See [Ika Environment Setup](./ika-environment.mdx) for instructions on forking the Ika repository, if necessary.

## Contribute via the Ika Improvement Proposal (SIP) process {#SIP}

The Ika Network is an open-source, decentralized, and permissionless protocol that welcomes community contributions. If you have an idea regarding a core protocol upgrade, the best way to make your voice heard is via submitting a Ika Improvement Proposal (SIP). For more information on SIPs, see [Contribute to Ika through SIPs](https://blog.ika.io/ika-improvement-proposals-sips/).
